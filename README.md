# 일기짠 (ilgijjan) — Web

> 아동을 위한 **AI 일기장 앱**의 웹 버전 프론트엔드

- 아이가 사진 한 장과 그날의 날씨·기분을 고르면, AI가 아이의 하루를 따뜻한 일기로 완성해 주는 서비스입니다.
- 완성된 일기는 나만 볼 수도, 다른 친구들에게 공개해 함께 나눌 수도 있습니다.

<br/>

## 📖 프로젝트 소개

- **무엇** : 아동용 AI 일기장 서비스의 웹 클라이언트
- **누구를 위해** : 글쓰기가 아직 서툰 아이들이, 고르고 누르는 것만으로 일기를 남길 수 있도록
- **백엔드** : Spring Boot로 구현된 API 서버와 연동
- **특징** : 휴대폰 사용 환경을 1순위로 고려한 **모바일 우선(Mobile-first)** UI

<br/>

## 🧩 이렇게 구현했어요

### 기능별 모듈 구조
도메인(기능) 단위로 폴더를 나눠, 관련된 API·컴포넌트·훅·타입을 한곳에 모았습니다.

```
src/
├── app/        # 라우터, 전역 설정 (queryClient, store)
├── features/   # 기능별 모듈
│   ├── auth/       # 로그인·인증
│   ├── diary/      # 일기 작성·조회·목록
│   ├── user/       # 유저 정보·설정
│   ├── like/       # 좋아요
│   └── storage/    # 이미지 업로드
├── pages/      # 화면(페이지) 단위 컴포넌트
└── shared/     # 공통 컴포넌트·훅·유틸
```

### 역할을 나눈 레이어 설계
한 파일이 모든 일을 떠안지 않도록, 책임을 단계로 분리했습니다.

```
Page (화면 조립)  →  Hook (비즈니스 로직)  →  Api (서버 통신)  →  Types (데이터 형태 정의)
```

- **Page** 는 화면을 조립만 하고, 로직은 두지 않습니다.
- **Hook** 에 데이터 처리·비즈니스 로직을 모읍니다. (`useDiaries`, `useCreateDiary` 등)
- **Api** 를 통해서만 서버와 통신합니다. (컴포넌트에서 직접 호출 금지)

### 상황에 맞춘 상태 관리 전략
상태의 성격에 따라 도구를 구분해 사용했습니다.

| 상태 종류 | 사용 도구 | 예시 |
|-----------|-----------|------|
| 서버에서 받아오는 데이터 | React Query | 일기 목록, 유저 정보 |
| 앱 전역 클라이언트 상태 | Jotai | 로그인 유저, 일기 작성 draft |
| 화면 안에서만 쓰는 상태 | useState | 모달 열림/닫힘, 탭 |
| 입력 폼 | React Hook Form | 이름 입력, 글쓰기 |

### 멀티스텝 일기 작성
일기 작성은 **사진 → 날씨 → 기분 → 글쓰기** 4단계로 진행됩니다.
각 단계의 입력은 Jotai에 임시 저장(draft)해 두었다가, 마지막 단계에서 **한 번에 서버로 전송**합니다.
AI가 일기를 생성하는 동안에는 상태를 주기적으로 확인(polling)하다가, 완성되면 결과 화면으로 이동합니다.

## 🔐 인증 방식

- 카카오 OAuth 기반 소셜 로그인을 사용합니다.
- 로그인 세션이 만료되면 자동으로 갱신되어, **사용자는 다시 로그인할 필요 없이** 서비스를 계속 이용할 수 있습니다.
